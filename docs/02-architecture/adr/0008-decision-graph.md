# 0008 — DecisionGraph and the structured-model → renderer pattern

- **Status:** Accepted
- **Date:** 2026-06-27
- **Deciders:** Lead Architect
- **Related:** [ADR-0007](0007-knowledge-synthesis.md)

## Context

ADRs could be generated by prompting an LLM with the artifacts. That would be the exact
*document → LLM → document* anti-pattern we rejected for the README: opaque, non-deterministic,
untestable. As the `KnowledgeGraph` becomes load-bearing, two things must be true: artifacts derive
from structured models, and those models can evolve without breaking existing renderers.

## Decision

1. **Every generated artifact follows `structured model → renderer`.** ADRs use:
   ```
   KnowledgeGraph → DecisionExtractor → DecisionGraph (typed Decision[]) → ADR Renderer
   ```
   No document→LLM→document anywhere in the core path.
2. **Version the graphs.** `KnowledgeGraph` and `DecisionGraph` carry `schema_version` (`v1`). The
   schema can grow (v2) additively without breaking renderers or stored projects.
3. **Typed entities, not dicts.** Decisions are a typed `Decision` (title, context, decision,
   alternatives, consequences, sources), not an untyped bag of fields.

## Alternatives considered

- **Prompt → ADR.** Fast. Rejected: opaque, non-deterministic, no provenance, can't be semantically
  tested.
- **Untyped extracted-field dictionary.** Flexible. Rejected (per review): retrofitting versioning
  and types after many renderers depend on it is far costlier than doing it now.

## Trade-offs

- (+) ADRs are deterministic, testable, and provenance-tracked — same guarantees as the README.
- (+) The pattern generalizes: architecture diagrams, roadmap, pitch deck are all future renderers.
- (+) Versioned schema = safe evolution; v2 can add `Component[]`, `Persona[]`, `Glossary` additively.
- (−) More upfront structure than free-form prompting.
- (−) Heuristic decision extraction is coarser than a frontier model's; an AI extraction pass is a
  future enhancement that keeps the renderer and provenance unchanged.

## Consequences

- The ADR artifact is a renderer over the `DecisionGraph` — the first decision is rendered today; a
  multi-ADR index follows naturally.
- The KnowledgeGraph is the shared semantic substrate for every artifact; this is the basis for
  later **bidirectional editing** (edit an artifact → update the graph → regenerate dependents).

## Future revisit

Grow the schema to v2 (Components, Personas, Constraints, Glossary), add an AI-assisted extraction
pass behind the same renderer, and add `graph.explain(entity)` for explainable provenance.
